SmartDoc AI

A fully local RAG system that lets you have an intelligent conversation with any PDF document — privately, instantly, and at zero cost.

Overview

The volume of information locked inside PDF documents is enormous — research papers, legal contracts, technical manuals, lecture notes. Finding a specific answer in a 100-page document is slow and frustrating. SmartDoc AI solves this.

Upload any PDF and ask questions about it in plain language. The system reads, understands, and retrieves exactly the relevant passages — then uses a local large language model to synthesise a precise, grounded answer. No cloud. No subscription. No data leaving your machine.

It was built as the final project for the Open Source Software Development course at Saigon University (Spring 2026), with a focus on building production-quality AI pipelines from entirely open-source components.

The Problem

Traditional keyword search fails to understand the intent behind a question. Cloud-based AI tools like ChatGPT require uploading sensitive documents to external servers — a privacy risk for legal, medical, or confidential content. And they cost money at scale.

The goal was to build a system that is accurate, private by default, free to run, and capable of understanding both English and Vietnamese — the two primary languages used in Vietnamese academic and professional settings.

How It Works — 5-Step Pipeline

SmartDoc AI is built around a classic RAG (Retrieval-Augmented Generation) architecture, with each component carefully chosen for multilingual quality and local performance.

1
Upload & Extract

The user uploads a PDF via the Streamlit interface. PDFPlumber parses the file, accurately extracting text while preserving layout — including tables and multi-column formats that simpler parsers often mangle.

2
Chunk & Embed

The extracted text is split into overlapping chunks of 1,000 characters with 100-character overlap using RecursiveCharacterTextSplitter — the overlap ensures no context is lost at chunk boundaries. Each chunk is then converted into a 768-dimensional vector by paraphrase-multilingual-mpnet-base-v2, a sentence-transformer model that natively supports 50+ languages including Vietnamese.

3
Index with FAISS

All vectors are indexed locally using FAISS (Facebook AI Similarity Search). FAISS enables millisecond-speed similarity search across millions of vectors without any external database or network dependency. The entire index lives in memory — fast, private, ephemeral.

4
Semantic Retrieval

When the user submits a question, it is embedded using the same MPNet model. FAISS then finds the top-3 chunks with the highest cosine similarity to the question vector. This is meaning-level matching — not keyword matching — so the system understands paraphrases and indirect questions correctly.

5
Generate & Respond

The retrieved chunks and the question are assembled into a structured prompt. A language detection step checks whether the question is in Vietnamese or English and selects the appropriate prompt template. The prompt is then passed to Qwen2.5:7b — running entirely locally via Ollama — which generates a concise, 3–4 sentence answer grounded in the retrieved context. If the document doesn't contain the answer, the model is instructed to say so rather than hallucinate.

System Architecture

Four layers, each with a single responsibility:

Presentation → Streamlit (file upload, Q&A interface) Application → LangChain (pipeline orchestration, prompt engineering) Data → FAISS vector store + MPNet embeddings (768-dim) Model → Qwen2.5:7b via Ollama (local LLM inference)

The separation of concerns means any layer can be swapped independently — for example, replacing Qwen2.5 with Llama 3 or Mistral requires changing a single line of configuration.

Key Highlights
100% Local & Private
Every component — embedding, vector search, LLM inference — runs on the user's own machine. Documents never leave the system. Ideal for sensitive content.
Zero API Cost
No OpenAI key, no monthly bill, no per-query charge. The entire stack is open-source: Qwen2.5:7b, sentence-transformers, FAISS, LangChain, Streamlit.
50+ Languages
The MPNet embedding model handles Vietnamese, English, Chinese, French, German, and 50+ other languages. The system auto-detects the question language and responds accordingly.
Honest by Design
Tested against out-of-context questions (e.g. asking a recipe PDF about calculus). The system is prompted to say "I don't know" rather than fabricate — a critical quality for real-world trust.
